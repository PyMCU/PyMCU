# Credits

## Richard Wardlow — Original pyMCU (2012)

Special thanks to **Richard Wardlow** ([@rwardlow](https://github.com/rwardlow)),
creator of the original [pyMCU](https://github.com/rwardlow/pyMCU) project in 2012.

The original pyMCU was a USB-connected PIC 16F1939 board that let you control
electronics from Python running on a host computer — a creative and accessible
bridge between software and hardware that inspired many makers and educators.

This project carries the PyMCU name with his gracious blessing. While the two
projects share a name and a mission to make microcontrollers approachable through
Python, they approach it from opposite ends:

| | Original pyMCU (2012) | This PyMCU |
|---|---|---|
| **Python runs on** | Host computer (PC/Mac/Linux) | Directly on the MCU |
| **MCU role** | Firmware managed by Richard's board | Bare-metal target for compiled code |
| **Model** | USB tether + Python library | AOT compiler — no runtime, no interpreter |
| **Goal** | Python <-> hardware bridge | Python as a first-class MCU language |

Richard's support, feedback, and generosity in ceding the name have been
invaluable to this project. Thank you.
